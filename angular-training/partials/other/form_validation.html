• Form element and its children have several useful properties to perform client validation:
    • myForm.email.$pristine: True if the user has not yet modified the element.
    • myForm.email.$dirty: True if the user has already modified the element.
    • myForm.email.$valid: True if the element validation has been successful.
    • myForm.email.$invalid: True if the element has validation errors.
    • myForm.email.$error: This object contains one property for each validation rule on the element. The value of the property is true if the rule has not been validated.
