Unit Testing E2E Testing
  • Code-level testing
  • Used to test classes/objects/services
  • Isolated/Sandboxed
  • Mocking required (angular-mocks.js)
  • Fast
  • Uses Jasmine/QUnit/Mocha
  • Application-level testing
  • Used for integration tests
  • Requires its own special web server
  • Can only test rendererd HTML
  • Slow
  • Uses AngularJS Scenario runner (angular-scenario.js), OR Protractor (with Selenium)
